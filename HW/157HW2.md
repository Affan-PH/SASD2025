**Code Smell: Incomplete Library Class** <cd>

- ความหมายของ code smell

Incomplete Library Class คือ code smell ที่เกิดขึ้นเมื่อเราใช้คลาสจากไลบรารีหรือคลาสที่มีอยู่แล้ว แต่คลาสนั้น ยังไม่ตอบโจทย์การใช้งานของเราแบบครบถ้วน
เช่น มี method ไม่พอ หรือพฤติกรรมบางอย่างที่เราต้องใช้แต่คลาสเดิมไม่มีให้ ทำให้โค้ดดูไม่ยืดหยุ่น และต้องหาทางแก้ไขเพิ่มเติมเอง <cd>

- ตัวอย่างโค้ดที่เกิด code smell <cd>

```class LibraryUser {
    public void borrowBook() {
        System.out.println("Borrow book");
    }
}
```

- ตัวอย่างโค้ดที่ refactoring แล้ว

วิธีแก้คือใช้ Inheritance หรือ Wrapper Class เพื่อเติมความสามารถที่ขาดไป โดยไม่ไปแก้โค้ดไลบรารีโดยตรง
```
class AdvancedLibraryUser extends LibraryUser {
    public void returnBook() {
        System.out.println("Return book");
    }
}
```

หลัง refactoring โค้ดจะใช้งานได้ครบขึ้น และยังไม่กระทบกับไลบรารีเดิม

- กรณีที่ code smell นี้ไม่จำเป็นต้อง refactoring

ถ้าคลาสจากไลบรารี ถูกออกแบบมาให้ใช้งานแค่นั้นอยู่แล้ว
และระบบของเรา ไม่ได้ต้องการฟังก์ชันเพิ่มเติมจริงๆ
เช่น ใช้แค่ borrowBook() อย่างเดียวตลอดทั้งโปรแกรม
กรณีนี้ถือว่าไม่จำเป็นต้อง refactoring เพราะโค้ดยังทำงานได้ตรงตามความต้องการ 
และไม่ทำให้ระบบซับซ้อนเกินไป
